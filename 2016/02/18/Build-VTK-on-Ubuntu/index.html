VTK is an open source framework for 3D computer graphics. I tried to build it on my Ubuntu systems, but failed for several times. I followed
and modified this scripts to build it successfully.

Qt Setup

VTK requires QT. I used Qt5 at first, but failed to install denali. I succeed with using QT4.

  • Download qt offline installer for linux:

  • Install to a separate directory, for example mine is installed at: $HOME/QT

  • Configure VTK with the following options

If CMake is not installed, You can install CMake by following Installing CMake. And If you have CMake errors, such as:

It’s a CMake missing modules directory discussed in Stackoverflow. Following below scripts to solve:

  • vi ~/.bashrc
  • edit .bahsrc by adding: export PATH=$PATH:/usr/bin:/bin
  • source ~/.bashrc

Another error is related to QT5, It shows that CMake cannot find my QT5 library.

Fixed it with following in /path/to/VTK/GUISupport/QT/CMakeLists.txt:

Methods to install Qt4, can be viewed here, remember to use “make install” after “make”.

  • mkdir qt-4.8.6-build && cd qt-4.8.6-build
  • wget http://download.qt-project.org/official_releases/qt/4.8/4.8.6/qt-everywhere-opensource-src-4.8.6.tar.gz
  • tar xzf qt-everywhere-opensource-src-4.8.6.tar.gz
  • cd qt-everywhere-opensource-src-4.8.6
  • make

Then in the VTK build directory, configure VTK with the path to the Qt build

Some errors such as:

>
Could NOT find Qt4 (missing: QT_QMAKE_EXECUTABLE QT_MOC_EXECUTABLE
QT_RCC_EXECUTABLE QT_UIC_EXECUTABLE QT_INCLUDE_DIR QT_LIBRARY_DIR
QT_QTCORE_LIBRARY)

Possible solutions:

  1. Add the path to the directory containing the qmake executable to your PATH environment variable (i.e. export PATH=/path/to/qmake/dir/:PATH under linux). Remember use source after configuration.
  2. Set the cmake variable QT_QMAKE_EXECUTABLE to the full path of the qmake executable (i.e. cmake /path/to/CMakeLists.txt -DQT_QMAKE_EXECUTABLE=/path/to/qmake/executable/dir/)
  3. Set the QT_QMAKE_EXECUTABLE in your CMakeCache.txt or using ccmake/cmakegui

Build

Once CMake finishes successfully configuring, use make inside the build directory.
