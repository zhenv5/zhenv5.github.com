heapq - Heap queue algorithm

heapq is a standard library in Python, documentation is here: heapq.

heapq maintains a min-heap. As we know min-heap is a binary tree for which parent node has a value less than or equal to any of its children. This implementation can use arrays. If the parent node’s index is k, the left child will be 2k+1, and the right child will be 2k+2, counting element from 0.

Functions

It provides following useful functions, detailed information can by found in the documentation.

  • heapq.heapify(x): transform list x into a heap, in-place, in linear time

  • heapq.push(heap,item): push the value item onto the heap, maintaining the heap invariant

  • heapq.pop(heap): pop and return the smallest item from the heap, maintaining the heap invariant. To access the smallest item without popping it, use heap[0]

  • heapq.pushpop(heap,item): push item onto the heap and return the smallest item from the heap. It runs more efficiently than heappush(heap,item) followed by a separate call to heappop(heap).

  • heapq.nlargest(n,iterableItems): return a list with the n largest elements from the dataset defined by iterableItems.

  • heapq.nsmallest(n,iterableItems): return a list with the smallest elements from the dataset defined by iterableItems.

Examples

Use Min-Heap to find the k largest(smllest) elements in an array

We can use heapq.nlargest() and heapq.nsmallest() functions to find the k largest/smallest elements in an iterable list.

What’s the idea behind these functions?

Procedures:

  • Build a Min Heap of the first k-elements (array[0] - array[k-1]). O(k)
  • Find the k-largest: for each element( after the k-th element in the array from k to n-1), compare it with the root: if the element is greater than the root then push this element onto this heap, else ignore it. O((n-k)log(k))
  • Find the k-smallest: for each element, compare it with the root, if the element is smaller than the root then push this element onto this heap, else ignore it. O((n-k)log(k))
  • Finally, heap has k largest/smallest elements and root of heap is the k-th largest/smallest element.

Time complexity: O((n-k)log(k) + k) without sorted output. If sorted output is needed then O(((n-k)log(k) + k )+ klog(k)) = O(nlog(k) + k).
